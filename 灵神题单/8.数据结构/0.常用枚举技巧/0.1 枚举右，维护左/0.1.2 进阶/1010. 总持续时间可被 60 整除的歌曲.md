抓虫现场：被遗忘的 60 的倍数
问题所在：当遇到的时间恰好是 60 的倍数时，你的匹配逻辑会落空。

我们来模拟一个会报错的场景，假设输入是 time = [60, 60]：

遍历第一个 60：

c % 60 是 0。

你去字典里找 60 - 0 = 60。显然字典里没有 60。

接着你把当前余数存入字典：mp[0] = 1。注意，你存进去的键是 0，不是 60！

遍历第二个 60：

c % 60 依然是 0。

你又去字典里找 60。字典里依然没有 60（只有刚才存的 0）。

严重后果：明明这两个 60 可以凑成 120（也就是 60 的倍数），但你的代码完美地错过了它们，最终返回 0！

最优雅的修正方法
当余数是 0 时，我们希望它去字典里找的另一半也是 0，而不是 60。
怎么把 60 变成 0，同时又不影响其他正常的数字（比如 20 找 40）呢？

答案是：再套一层求余运算！

把你的 (60 - (c % 60)) 改成 (60 - (c % 60)) % 60：

如果 c = 20：(60 - 20) % 60 = 40 % 60 = 40。（正常匹配）

如果 c = 60：(60 - 0) % 60 = 60 % 60 = 0。（完美匹配到存入字典的 0！）

```python
class Solution:
    def numPairsDivisibleBy60(self, time: List[int]) -> int:
        res = 0
        mp = defaultdict(int)
        for i,c in enumerate(time):
            if (60 - (c % 60))%60 in mp:
                res += mp[(60 - (c % 60))%60]
            mp[c % 60] += 1

        return res
```
