这题必须将翻转后的数字作为哈希表的key，如果直接将原数字作key的话，遍历到的数字翻转后的前导零是查不到的。

根本原因是题目要求的是 reverse(nums[i]) == nums[j]，不是 reverse(nums[j]) == nums[i]，i<j 必须满足，左右不是对称的。

```python
class Solution:
    def minMirrorPairDistance(self, nums: List[int]) -> int:

        res = inf
        def getReverse(num):
            # num = list(str(num))[::-1]
            # i = 0
            # while nums[i] == 0 and i < len(num) - 1:
            #     i += 1
            # num = num[i:]
            # return int(''.join(num))
            return int(str(num)[::-1])
        mp = defaultdict(int)
        for i,c in enumerate(nums):
            # 这里用本身判断是否存在，然后存入反转后的结果，有说法
            if c in mp:
                res = min(res,i - mp[c])
            t = getReverse(c)
            mp[t] = i

        return res if res != inf else -1
```
