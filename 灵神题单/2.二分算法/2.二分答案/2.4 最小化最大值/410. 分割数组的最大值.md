l如果初始化为-1错误的原因：

你的 check 函数里缺少了一个常识性的物理防御：如果有一个数字本身就比箱子的容量 mid 还要大，那是绝对装不进去的！

我们来模拟一个会让你代码翻车的极端场景：
假设 nums = [2, 10]，k = 2。
如果二分法猜的 mid = 5（意味着每个子数组的和最大只能是 5）。

遇到 2：0 + 2 <= 5，装入当前数组，s = 2。

遇到 10：2 + 10 <= 5 为 False。进入 else。

你的 else 逻辑判断 cnt (1) >= k (2) 为 False。于是 cnt += 1 变成 2，然后强行把 10 塞给了新的子数组，s = 10。

循环结束，返回 True！

严重后果：系统认为 mid = 5 是一个合法的答案！但实际上，数组里明明有个 10，你不可能把它塞进容量为 5 的子数组里。最终二分出来的答案会偏小。

最优雅的修复方式：提高你的“下限”！
解决这个问题有两种方法：

第一种是在 check 里面加一句 if c > mid: return False。

第二种（最优雅、速度最快的方法）：直接修改左边界 l 的初始值！

既然子数组的容量绝对不可能小于数组中的最大元素（否则最大的那个数永远放不进去），那我们的下界就不该是 -1，而应该是 max(nums) - 1！

只要让二分查找的起始点高于最大元素，check 函数里的那个 Bug 就永远不会被触发了。

```python
class Solution:
    def splitArray(self, nums: List[int], k: int) -> int:
        n = len(nums)

        # 二分答案的思路都是逆天思路
        def check(mid,nums,k):
            cnt = 1
            s = 0
            for c in nums:
                if s + c <= mid:
                    s += c
                else :
                    if cnt >= k:
                        return False
                    cnt += 1
                    s = c
            return True

        l,r = max(nums) - 1,sum(nums) + 1
        while l + 1 <r :
            mid = (l + r) // 2
            if check(mid,nums,k):
                r = mid
            else :
                l = mid

        return r
```
